# EazyMake 0.9.10 — 代码质量重构

---

## 1 背景

0.9.9 完成了安装钩子 Lua 化，项目到达"零技术债务"的功能层面。但在代码审查中识别出以下结构性问题，属于实现层面的技术债务：

| 问题 | 严重性 | 位置 |
|---|---|---|
| `run_install_hook_script()` 与 `run_hook_script()` ~70 行重复 | 中 | `src/lua_api.cpp` |
| `run_install_script()` 参数膨胀至 9 个 | 低 | `src/pkg.cpp` |
| Lua 栈状态隐式契约（`register_api` 调用前后） | 低 | `src/lua_api.cpp` |
| `detect_install_script()` 因 `static` 不可测 | 低 | `src/pkg.cpp` |

本版本**只重构、不新增功能、不改变外部行为**。

---

## 2 目标

1. **消除 `run_install_hook_script` 与 `run_hook_script` 的重复代码** — 提取通用 sandbox 执行/ctx 构建框架
2. **压缩 `run_install_script()` 参数** — 引入 `InstallHookContext` 结构体
3. **加固 Lua 栈安全** — `register_api()` 调用点增加栈位置断言
4. **提高可测试性** — `detect_install_script()` 移至 `pkg.hpp` 并添加单元测试

---

## 3 详细设计

### 3.1 提取通用 sandbox 执行框架

两个函数的共同流程：

```
注册 ezmk API → 构建 sandbox → 加载脚本 → lua_pcall(执行chunk) → 查找 run() → 构建 ctx → lua_pcall(run) → 取返回值
```

**方案**：新增内部函数 `run_lua_script_with_ctx()`，接收一个 ctx 构建回调：

```cpp
// src/lua_api.cpp（内部函数，不在头文件中声明）

// Callback type: builds the ctx table on the Lua stack and returns the number
// of fields pushed. The ctx table itself is at the top of the stack on entry.
using BuildCtxFn = std::function<void(lua_State* L)>;

// Shared sandbox execution pipeline. Handles:
//   1. API registration  2. Sandbox construction  3. Script loading
//   4. Chunk execution   5. run() lookup           6. ctx building (via callback)
//   7. run(ctx) call     8. Exit code extraction
// Returns the exit code from run(), or 1 on error.
static int run_lua_script_with_ctx(lua_State* L,
                                    const fs::path& script_path,
                                    const fs::path& pkg_root_dir,
                                    BuildCtxFn build_ctx);
```

两个公开函数退化为薄封装：

```cpp
int run_hook_script(lua_State* L, const fs::path& script_path,
                    const fs::path& output,
                    const fs::path& project_root,
                    const std::string& profile) {
    g_current_script_pkg_root = ...;  // 设置上下文
    return run_lua_script_with_ctx(L, script_path, project_root, [&](lua_State* L) {
        lua_pushstring(L, output.string().c_str());
        lua_setfield(L, -2, "output");
        lua_pushstring(L, project_root.string().c_str());
        lua_setfield(L, -2, "project_root");
        lua_pushstring(L, profile.c_str());
        lua_setfield(L, -2, "profile");
    });
}

int run_install_hook_script(lua_State* L, const fs::path& script_path,
                             const std::string& pkg_name,
                             const fs::path& pkg_root,
                             const fs::path& install_path,
                             const std::string& scope) {
    g_current_script_pkg_root = pkg_root;
    return run_lua_script_with_ctx(L, script_path, pkg_root, [&](lua_State* L) {
        // ... 6 个 ctx 字段
    });
}
```

### 3.2 压缩 `run_install_script()` 参数

```cpp
// include/ezmk/pkg.hpp 或 pkg.cpp 内部
struct InstallHookContext {
    std::string pkg_name;
    fs::path    pkg_root;
    fs::path    install_path;
    std::string scope;       // "project" / "user" / "global"
};

// 新签名（9 → 6 参数）
static bool run_install_script(const fs::path& script, const fs::path& cwd,
                                bool assume_yes, std::string_view label,
                                bool is_lua,
                                const InstallHookContext& hook_ctx);
```

`scope_to_string()` 已在 0.9.9 收尾中提取，无需重复。

### 3.3 Lua 栈位置断言

在 `register_api(L, ...)` 调用前后增加 debug 断言：

```cpp
int top_before = lua_gettop(L);
register_api(L, pkg_root);
assert(lua_gettop(L) == top_before && "register_api must not change stack size");
```

> `assert()` 在 release build 中编译为空，无性能影响；debug build 中可捕获栈泄漏。

仅在 `run_lua_script_with_ctx()` 内部做一处即可覆盖所有调用路径。

### 3.4 `detect_install_script()` 可测试化

**方案 A（推荐）**：移至 `pkg.hpp` 作为公开函数，添加单元测试。

**方案 B**：保持 `static`，通过集成测试间接覆盖。但集成测试需要构造真实的包目录结构和脚本文件，成本较高。

选择方案 A。函数已足够稳定（简单的文件存在性检查），暴露为公开 API 风险低。

```cpp
// include/ezmk/pkg.hpp 新增声明
fs::path detect_install_script(const fs::path& pkg_root,
                                std::string_view basename);
```

测试用例（新增 3 个）：
- `.lua` 优先于 `.ps1`（Windows）或 `.sh`（Linux）
- 仅 `.lua` 存在 → 返回 `.lua`
- 无脚本 → 返回空路径
