# EazyMake 0.9.5 — 跨平台体验与质量保障

> 所有条目均已在对应版本中实现。

---

## 1 背景

0.9.4 完成了文档和质量打磨。本版本聚焦两个 temp.md 揭示的缺口：**Windows 原生安装体验**（当前仅有 bash 脚本，裸 Windows 用户需手动下载预编译二进制）和**缺乏端到端集成测试**（17 个测试文件全部为单元测试，从未真正验证完整的 build pipeline）。

作为 1.0.0 之前的质量保障版本，三平台冒烟测试将确保 Linux / macOS / Windows 上的行为一致性。

---

## 2 目标

1. **PowerShell 安装脚本** — 新建 `install.ps1`，对标 `install.sh` 功能，提供 Windows 原生一键安装体验
2. **集成测试** — 至少 1 个端到端测试场景：`project new` → `pkg install` → `project build` → `project run`
3. **三平台冒烟测试** — Linux + macOS + Windows/MSYS2 上验证核心流程

---

## 3 详细设计

### 3.1 PowerShell 安装脚本 (`install.ps1`)

**对标 `install.sh` 的核心功能**：

| 功能       | `install.sh`                                      | `install.ps1`                                                 |
| ---------- | ------------------------------------------------- | ------------------------------------------------------------- |
| 平台检测   | Linux/macOS/MSYS2                                 | Windows (PowerShell 5.1+)                                     |
| 二进制下载 | `curl` 从 GitHub Release 下载                     | `Invoke-WebRequest` 从 GitHub Release 下载                    |
| 安装路径   | `~/.local/bin` (Linux) / `/usr/local/bin` (macOS) | `$env:LOCALAPPDATA\ezmk\bin`                                  |
| PATH 配置  | shell rc 文件追加                                 | 用户 PATH 环境变量（`[Environment]::SetEnvironmentVariable`） |
| 仓库预注册 | `ezmk repo add` 官方仓库                          | 同                                                            |
| 权限检查   | `sudo` 检测                                       | 管理员权限检测 + UAC 提示                                     |
| 校验       | SHA-256 校验下载文件                              | 同                                                            |
| 幂等性     | 重复运行安全（覆盖升级）                          | 同                                                            |

**脚本结构**：
```powershell
# install.ps1
param(
    [string]$Version = "latest",
    [string]$InstallDir = "$env:LOCALAPPDATA\ezmk",
    [switch]$NoPath = $false,
    [switch]$DryRun = $false
)

# 1. 平台检测（确认 Windows）
# 2. 下载 ezmk.exe（GitHub Release API）
# 3. SHA-256 校验
# 4. 安装到目标目录
# 5. 注册 PATH
# 6. 预注册官方仓库
# 7. 验证安装（ezmk --version）
```

**安装方式**：
```powershell
# 方式一：下载脚本后运行
.\install.ps1

# 方式二：一键远程执行（类似 curl | bash）
irm https://raw.githubusercontent.com/.../install.ps1 | iex
```

**安全考量**：
- 推荐方式一（审查后运行），方式二仅为便利提供
- 默认使用 TLS 1.2+
- 下载文件 SHA-256 校验

### 3.2 集成测试

**测试框架**: 在现有 Catch2 框架内新增 `test/test_integration.cpp`，使用 `ezmk` 二进制（先构建）进行端到端测试。

**测试场景**：

```
场景：从零创建项目 → 安装依赖 → 构建 → 运行
  1. ezmk project new test_app
  2. 编辑 ezmk.toml，添加 [depends] lib = ["catch2"]
  3. ezmk pkg install catch2 -p -y
  4. ezmk project build
  5. 验证 build/ 目录存在可执行文件
  6. ezmk project run
  7. 验证 stdout 包含预期输出

场景：增量构建缓存命中
  1. ezmk project build（首次）
  2. ezmk project build（二次，应全部 cache hit）
  3. 验证日志输出包含 "cache hit" / "up to date"

场景：Watch 模式启动 → 文件变更触发重编译
  1. ezmk project watch（后台启动，超时终止）
  2. 修改源文件
  3. 验证触发了重编译

场景：交叉验证 (GCC ↔ MSVC)
  1. 同一项目在 GCC 下构建通过
  2. 同一项目在 MSVC 下构建通过（若环境可用）
```

**实施要点**：
- 利用 Catch2 `TEST_CASE` + `SECTION` 组织
- 使用 `util::run_command()` 调用 `ezmk` 二进制
- 每个场景使用独立临时目录（`std::filesystem::temp_directory_path`）
- 集成测试标记为 `[integration]` tag，允许按需运行

### 3.3 三平台冒烟测试

**测试矩阵**：

| 平台    | 环境                                        | 测试项                                  |
| ------- | ------------------------------------------- | --------------------------------------- |
| Linux   | Ubuntu 22.04+ / g++                         | install.sh → project new → build → run  |
| macOS   | macOS 13+ (Intel + Apple Silicon) / clang++ | install.sh → project new → build → run  |
| Windows | Windows 10+ / MSYS2 (g++)                   | install.sh → project new → build → run  |
| Windows | Windows 10+ / MSVC (cl.exe)                 | install.ps1 → project new → build → run |

**每平台验证步骤**：
1. 全新安装（`install.sh` 或 `install.ps1`）
2. `ezmk --version` 输出版本号
3. `ezmk --help` 输出帮助信息
4. `ezmk project new smoke_test`
5. `cd smoke_test && ezmk project build`
6. `ezmk project run`
7. `ezmk pkg install catch2 -p -y`（验证网络 + 包管理）
8. 修改 `ezmk.toml` 添加 `[depends] lib = ["catch2"]`，重新 `build`（验证依赖编译）
9. `ezmk utils cc`（验证内置工具）
