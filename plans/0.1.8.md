# EazyMake 0.1.8 — 跨平台支持与编译器探测

---

## 1 现状

当前 `config::parse_language()` 将语言字符串硬编码映射到编译器名（`"C++"` → `"g++"`、`"C"` → `"gcc"`），`build::find_compiler()` 仅在 Windows 上追加 `.exe` 后缀。如果用户没有安装 g++，错误信息是通用的 "compiler not found" + 安装指引，但不会自动寻找替代编译器。

涉及文件：
- `include/ezmk/config.hpp` — `LanguageInfo` 结构体（`compiler` 字段）
- `src/config.cpp` — `parse_language()` 函数（硬编码 g++/gcc）
- `src/build.cpp` — `find_compiler()` 函数（仅追加 .exe）、`build_project()` 中的编译器验证逻辑
- `include/ezmk/util.hpp` — `EZMK_WIN` / `EZMK_EXE_SUFFIX` / `EZMK_OBJ_SUFFIX` 宏定义（缺少 macOS 检测）

---

## 2 目标行为

编译器选择优先级（从高到低）：

1. **环境变量覆盖**：`CXX`（C++）/ `CC`（C）—— 用户显式指定时直接使用
2. **平台默认**（按顺序探测，取第一个可用者）：
   - **Linux**：`g++` → `clang++` → `c++`
   - **macOS**：`g++` → `clang++` → `c++`
     - 注意：macOS 上 `g++` 可能是 Xcode CLT 安装的 clang 别名
   - **Windows（MSYS2/MinGW）**：`g++` → `clang++`
   - **Windows（原生）**：`cl` → `clang-cl`（仅探测，不实现 MSVC 编译流程，见 0.2.1）
3. **全找不到**：给出清晰的错误信息和安装指引（按当前平台显示对应的安装命令）

### C 编译器的对应列表

| C++ 编译器 | C 编译器 |
|-----------|---------|
| `g++`     | `gcc`   |
| `clang++` | `clang` |
| `c++`     | `cc`    |

### 3.2 编译器探测实现

- [ ] 在 `include/ezmk/build.hpp` 中声明新函数：
  ```cpp
  // Detect the best available C/C++ compiler. Respects $CXX/$CC env vars.
  // Returns the compiler executable name (e.g. "g++", "/usr/bin/clang++").
  // language: "C++" or "C" (from config::LanguageInfo context).
  // Throws fatal_error if no compiler is found.
  std::string detect_compiler(const std::string& language);
  ```
- [ ] 在 `src/build.cpp` 中实现 `detect_compiler()`：

  ```cpp
  std::string detect_compiler(const std::string& language) {
      bool is_cxx = (language == "C++");
      
      // 1. Check environment variable override
      const char* env = is_cxx ? std::getenv("CXX") : std::getenv("CC");
      if (env && env[0] != '\0') {
          // Verify the env-specified compiler is usable
          std::string candidate(env);
          auto res = util::run_command(candidate + " --version 2>&1");
          if (res.exit_code == 0) return candidate;
          util::warn(std::string("$") + (is_cxx ? "CXX" : "CC") +
                     " is set to '" + candidate + "' but it is not executable");
      }
      
      // 2. Platform-specific candidate list
      std::vector<std::string> candidates;
  #ifdef EZMK_WIN
      // Check for MSVC compiler (cl.exe)
      {
          auto res = util::run_command("cl 2>&1");
          if (res.exit_code == 0) {
              util::info("MSVC compiler (cl.exe) detected. "
                         "Full MSVC support is planned for v0.2.1. "
                         "Using g++ instead.");
          }
      }
      candidates = is_cxx
          ? std::vector<std::string>{"g++", "clang++"}
          : std::vector<std::string>{"gcc", "clang"};
  #elif defined(EZMK_MACOS)
      candidates = is_cxx
          ? std::vector<std::string>{"g++", "clang++", "c++"}
          : std::vector<std::string>{"gcc", "clang", "cc"};
  #else // Linux
      candidates = is_cxx
          ? std::vector<std::string>{"g++", "clang++", "c++"}
          : std::vector<std::string>{"gcc", "clang", "cc"};
  #endif
      
      // 3. Probe each candidate
      for (auto& c : candidates) {
          auto res = util::run_command(c + " --version 2>&1");
          if (res.exit_code == 0) {
              // Verify compiler supports the language standard (optional but recommended)
              // by compiling an empty file: echo "" | g++ -std=c++17 -x c++ -E -
              return c;
          }
      }
      
      // 4. None found — fatal with install instructions
      std::string msg = "no C";
      msg += (is_cxx ? "++" : "");
      msg += " compiler found.\n\n";
  #ifdef EZMK_WIN
      msg += "  Install MSYS2: https://www.msys2.org/\n";
      msg += "  Then: pacman -S mingw-w64-x86_64-gcc";
  #elif defined(EZMK_MACOS)
      msg += "  Option A: xcode-select --install  (Apple Clang)\n";
      msg += "  Option B: brew install gcc          (GNU GCC)";
  #else
      msg += "  Debian/Ubuntu: sudo apt install g++\n";
      msg += "  RHEL/Fedora:   sudo dnf install gcc-c++\n";
      msg += "  Arch:          sudo pacman -S gcc";
  #endif
      util::fatal(msg);
      return {}; // unreachable
  }
  ```

## 5 注意事项

- 编译器探测结果应缓存（进程生命周期内只探测一次），避免每次调用重复执行 `--version`
- 探测仅在 `project build` / `project run` / `pkg install`（编译包时）触发，不影响 `project new` 等其他命令
- macOS 上 `g++` 可能实际指向 `clang++`（Xcode CLT），探测时需验证其确实是 GCC 还是 Clang 别名，但行为上两种都接受
  - 可通过 `g++ --version` 的输出来区分（含 "Apple" 或 "clang" → 实际是 Apple Clang）
  - Apple Clang 的 `-std=c++17` 支持完整，可直接使用
- 需要同时探测对应标准库支持（例如 `g++ -std=c++17` 是否被接受），可在探测时用 `-std=c++17 -E - < /dev/null` 验证
  - Windows 上用 `echo. | g++ -std=c++17 -x c++ -E -` 代替
- `EZMK_OBJ_SUFFIX` 当前定义为 `.obj`（Windows）和 `.o`（其他），但 MinGW g++ 实际输出 `.o`——需在 MSYS2 上验证并修正
- `cache.cpp` 中 `compile_sources()` 使用 `replace_extension(EZMK_OBJ_SUFFIX)` 来生成目标文件路径，后缀修正后缓存路径也会变化
- 编译器探测不改变 `cache.cpp` 中的缓存逻辑（`record.json` 中 `compiler` 字段已记录编译器名，不同编译器不会共享缓存）
- 如果用户通过 `$CXX` 覆盖为 `clang++`，`.d` 依赖文件格式仍然兼容（Clang 使用与 GCC 相同的 `-MMD -MF` 格式）
- 对于 MinGW 用户，`-lwinhttp` 是必需的（用于 HTTP 下载）——`build.sh` 已在 MSYS2 平台上自动添加
- 对于 macOS 用户，`-static` 链接标志可能导致警告（macOS 不完全支持静态链接），需改为 `-static` 仅在 Linux/MinGW 上使用
- 不同编译器的 warning flags 可能不同（如 `-Wall -Wextra` 是 GCC/Clang 通用，但 MSVC 使用 `/W4`），本版本不处理 MSVC flags
