# EazyMake 0.9.0 — 准备发布正式版

---

## 1 背景

`EazyMake` 经过约一个月的测试性开发（0.1.x ~ 0.2.x），已具备稳定的跨平台构建、包管理、仓库、Lua 工具链、i18n 与 shell 补全能力，可以准备发布首个面向用户的正式版。

本版本**不新增核心功能**，聚焦"让别人能顺利装上、看懂、上手"三件事：

1. **一键安装脚本**（`install.sh`，`curl | bash`）
2. **文档整理**（`docs/cli.md`、安全性集中化、README 双语互链）
3. **基本教程**（`tutorial/`）

> 与源码相关的既有事实：构建入口是 `bash build.sh`（生成 locale 数据 + 版本头再编译）；locale 在**编译期**嵌入二进制（`scripts/embed_locale.py` → `src/locale_data.cpp`），因此运行时**只需分发 `ezmk` 可执行文件**，无需附带 `locale/`。zsh 补全脚本为 `completions/_ezmk`。

## 2 自动安装脚本 `install.sh`

### 2.1 目标

- Linux / macOS / MSYS2 环境：`curl -fsSL <script_url> | bash` 一键安装
- 裸 Windows（非 MSYS2）环境：继续用 GitHub Release 分发预编译二进制（本脚本不覆盖）
- 幂等、可重入、失败即清理，`set -euo pipefail`

### 2.2 安装流程

1. **环境自检**（缺失即报错退出，给出安装建议）：
   - 必需：`git`、`bash`、C++ 编译器（`g++` 或 `clang++`，C++17）
   - 构建期：`python3`（或 `python`）——缺失时 `build.sh` 会退化为英文兜底 locale，脚本应**警告但继续**
   - 识别平台：`uname -s`（`Linux` / `Darwin` / `MINGW*|MSYS*`）
2. **申请临时目录**：`WORK="$(mktemp -d)"`，注册 `trap 'rm -rf "$WORK"' EXIT` 保证清理
3. **克隆项目**：`git clone --depth 1 https://github.com/3667808244/EazyMake.git "$WORK/EazyMake"`
   - 支持 `EZMK_REF` 环境变量指定 tag/branch（默认 clone 默认分支；发布后建议指向最新 release tag）
4. **构建**：`cd "$WORK/EazyMake" && EZMK_VERSION="<resolved>" bash build.sh`
   - `<resolved>` 取自 `git describe --tags --always`，避免 `build.sh` 里写死的 `EZMK_VERSION` 默认值（当前为陈旧的 `0.2.4`，见 §2.5）
5. **安装二进制**：
   - `install -Dm755 build/ezmk "$PREFIX/bin/ezmk"`（`PREFIX` 默认 `$HOME/.local`，可用 `PREFIX` 环境变量覆盖）
   - MSYS2 下产物为 `build/ezmk.exe`，需按平台选文件名
6. **PATH 检查**：若 `$PREFIX/bin` 不在 `PATH`，打印提示（并可选地追加到对应 shell rc）
7. **zsh 补全安装**（若检测到 zsh，见 §2.4）：
   - `mkdir -p ~/.zsh/completions && cp completions/_ezmk ~/.zsh/completions/`
   - 幂等追加 `fpath=(~/.zsh/completions $fpath)` 与 `autoload -Uz compinit && compinit` 到 `~/.zshrc`（先 grep 去重，避免重复行）
8. **清理**：`trap` 自动删除临时目录
9. **收尾**：打印 `ezmk version` 输出与"重启终端或 `source ~/.zshrc`"提示

### 2.3 `curl | bash` 的健壮性要点

- **stdin 被占用**：`curl | bash` 时脚本自身占据 stdin，任何交互提示必须从 `/dev/tty` 读（`read ... < /dev/tty`），否则会读到脚本内容。**首选无交互**：用环境变量（`PREFIX`、`EZMK_REF`、`EZMK_NO_COMPLETIONS`）替代提问。
- **原子性**：先装到临时名再 `mv` 到最终路径，避免中途失败留下半个二进制。
- **可覆盖变量**（文档化）：`PREFIX`、`EZMK_REF`、`EZMK_VERSION`、`CXX`/`CC`/`CXXFLAGS`（透传给 `build.sh`）、`EZMK_NO_COMPLETIONS=1`（跳过补全）。
- **不需要 root**：默认装到 `$HOME/.local`，避免 `sudo`；如需系统级由用户显式设 `PREFIX=/usr/local`。

### 2.4 zsh 检测的纠正

原计划用 `chsh -l` 判断是否有 zsh——**不可靠**：`chsh -l` 在 macOS/BSD 上不支持该 flag，且它列的是"可选登录 shell"而非"是否安装 zsh"。改为：

- 主判据：`command -v zsh`（zsh 可执行是否存在）
- 辅助：读 `/etc/shells` 确认（可选）
- 补全目录沿用计划的 `~/.zsh/completions`（与 `completions/_ezmk` 头部注释里的用户级安装说明一致）

> 备选（更规范）：若存在 `${fpath}` 标准站点目录 `/usr/share/zsh/site-functions/` 且可写，则装到那里，免改 `.zshrc`。默认仍走用户级 `~/.zsh/completions` 以免权限问题。

### 2.5 `build.sh` 配套修正

- **版本默认值陈旧**：`build.sh` 里 `EZMK_VERSION="${EZMK_VERSION:-0.2.4}"`。发布时应：
  - 让 `install.sh` 显式传入 `EZMK_VERSION`（来自 `git describe`）；
  - 并把 `build.sh` 的兜底默认值更新为当前发布版本，避免本地直接 `bash build.sh` 得到错误版本号。
- **可选**：`build.sh` 增加 `--prefix` / `install` 目标，或新增独立 `make install`，让本地构建也能一键安装（与 `install.sh` 复用同一套补全安装逻辑）。

### 2.6 README 安装命令

在 `README.md` 与 `README_ZH.md` 的 Quick start 顶部加"一行安装"：

```bash
curl -fsSL https://raw.githubusercontent.com/3667808244/EazyMake/main/install.sh | bash
```

并注明：裸 Windows 用户请从 GitHub Release 下载预编译 `ezmk.exe`；自定义安装位置用 `PREFIX=/usr/local ... | bash`。

## 4 基本教程

### 4.1 目标

面向新用户的 `tutorial/`，从零到跑通一个多文件 + 依赖包的项目。建议章节：

1. **安装与验证**（`install.sh` / `ezmk version`）
2. **第一个项目**：`ezmk project new hello` → 项目结构讲解（`ezmk.toml`、`src/`、`include/`）→ `build` / `run`
3. **理解 `ezmk.toml`**：`[project]` / `[compile]` / `[link]` / `[depends]`，语言与类型
4. **增量构建与缓存**：改一个文件观察只重编该文件（`--verbose` 看缓存命中）
5. **构建 Profile 与并行**：`--profile debug`、`-j`
6. **用包**：`repo add` → `repo update` → `pkg install` → 在 `[depends]` 中引用
7. **Watch 与 Hooks**：`project watch`、`[hooks]` 小例
8. **Utils 工具**：`ezmk utils cc` 生成 `compile_commands.json`（clangd 集成）

每章配可运行的最小示例目录。教程与 `docs/`（规范）分工：`tutorial/` 讲"怎么上手"，`docs/` 讲"精确定义"。

## 6 注意事项

- **依赖顺序**：本版本的 i18n 相关发布质量依赖 **0.2.6** 先修复 `{???}`（见 `plans/0.2.6.md`）。建议 0.2.6 合入后再发 0.9.0。
- **`curl | bash` 的信任问题**：文档应同时给出"先下载审阅再执行"的两步用法（`curl -fsSL ... -o install.sh; less install.sh; bash install.sh`），尊重安全谨慎的用户。
- **locale 无需分发**：locale 编译期嵌入，安装只拷贝二进制即可；勿在 `install.sh` 里画蛇添足拷贝 `locale/`（运行时 `../locale/` 覆盖机制是可选的高级用法）。
- **macOS 静态链接限制**：`build.sh` 在 Darwin 下用动态链接（已处理），`install.sh` 无需特殊处理，但教程/文档应说明 macOS 产物为动态链接。
- **`build.sh` 版本默认值必须修**：否则本地 `bash build.sh` 出来的二进制 `ezmk version` 会报陈旧的 `0.2.4`，发布前必须纠正（§2.5）。

## 7 开放问题

1. **是否引入 `make install` / Makefile**：`install.sh` 与 `build.sh` 已覆盖，Makefile 是否多余？倾向仅保留 shell 脚本，除非有打包需求（deb/rpm/homebrew）。
2. **Homebrew / scoop / AUR 分发**：是否在 0.9.0 就提交到包管理器？建议 0.9.0 先稳住 `install.sh` + Release，第三方分发留待社区或后续。
3. **补全的系统级安装**：默认用户级（`~/.zsh/completions`）以免权限问题；是否在可写时优先 `site-functions`？见 §2.4 备选。
4. **教程是否双语**：`tutorial/` 先出一种语言（跟随主 README），另一种语言留待后续或社区。
