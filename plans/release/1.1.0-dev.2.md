# EazyMake 1.1.0-dev.2 — 包开发体验提升

---

## 1 背景

0.9.7 实现了 `ezmk.toml` 的 `precompiled` 字段（第一层：单平台预编译包支持），用户可以将外部 CMake/make 构建好的 `.a` 打包分发。但当前缺少：

- **多平台共包分发**：同一个 tar.gz 里放 Win/Lin/mac 三平台的 `.a`，安装时自动选择
- **`index.toml` 平台映射**：仓库索引层按 `os_arch_toolchain` 过滤，用户 `pkg install` 时自动匹配当前平台
- **打包命令**：尚无 `ezmk project pack` 子命令，打包流程仍为手动

---

## 2 交付物

| # | 交付物 | 优先级 | 说明 |
|---|--------|--------|------|
| 1 | **多平台共包支持**（第二层） | P0 | `lib/` 下按命名约定 `lib<name>.<os>-<arch>.a` 放多平台产物，安装时自动匹配当前平台 |
| 2 | **`index.toml` 平台映射**（第三层） | P0 | `[[packages]]` 新增 `platform` 字段（`os_arch_toolchain` 三元组），repo 搜索时自动过滤 |
| 3 | **`ezmk project pack`** | P1 | 新增打包子命令，将 `static` 项目一键打包为标准 EazyMake 包格式 |

## 3 详细设计

### 3.1 多平台共包（第二层）

**命名约定**：`lib<name>.<os>-<arch>.a`

```
sdl2/
├── ezmk.toml
├── include/
└── lib/
    ├── libsdl2.win-x64.a
    ├── libsdl2.linux-x64.a
    └── libsdl2.mac-arm64.a
```

**平台标识**（对齐 Rust target triple 的简化版）：

| OS | Arch | 标识 |
|----|------|------|
| Windows | x86_64 | `win-x64` |
| Windows | x86 | `win-x86` |
| Linux | x86_64 | `linux-x64` |
| Linux | aarch64 | `linux-arm64` |
| macOS | x86_64 | `mac-x64` |
| macOS | aarch64 | `mac-arm64` |

**选择逻辑**：
1. 检测当前平台 → `util::detect_platform_tag()` 返回如 `win-x64`
2. 遍历 `lib/` 下的文件，匹配 `<name>.<tag>.a`
3. 精确匹配优先 → fallback 到无后缀 `lib<name>.a`（向后兼容第一层）
4. 无匹配 → 报错，列出可用的平台

### 3.2 `index.toml` 平台映射（第三层）

在 `[[packages]]` 条目中新增 `platform` 字段：

```toml
[[packages]]
name = "sdl2"
version = "2.30.12"
file = "packages/sdl2-2.30.12-win-x64.tar.gz"
platform = "win-x64"
sha256 = "..."

[[packages]]
name = "sdl2"
version = "2.30.12"
file = "packages/sdl2-2.30.12-linux-x64.tar.gz"
platform = "linux-x64"
sha256 = "..."

[[packages]]
name = "sdl2"
version = "2.30.12"
file = "packages/sdl2-2.30.12-mac-arm64.tar.gz"
platform = "mac-arm64"
sha256 = "..."
```

**搜索过滤**（`repo::search_package()`）：
1. 检测当前平台 tag
2. 在 `[[packages]]` 中过滤：`platform` 匹配当前平台 或 `platform` 为空（向后兼容）
3. 在匹配的候选中选最高版本
4. 无匹配 → 报错 "package {name} has no build for {platform}"

**`index.toml` 格式兼容性**：
- `platform` 字段可选，缺失时视为"全平台可用"（向后兼容所有现有包）
- 同一 `name + version + platform` 组合只能有一条

### 3.3 `ezmk project pack`（P1）

```
ezmk project pack [--output <dir>]
```

将当前 `type = "static"` 项目打包为 `.tar.gz`：
1. 验证项目类型为 `static`
2. 编译项目（如未构建）
3. 收集 `include/` 头文件 + `build/lib<name>.a` + `ezmk.toml`
4. 打包为 `<name>-<version>.tar.gz`
5. 计算并打印 SHA-256
6. 可选：自动生成 `index.toml` 条目

## 5 关键设计决策

- **平台标识**：采用简化版 triplet `os-arch`，不包含 toolchain（预编译包不与工具链绑定）
- **向后兼容**：`platform` 字段可选，第一层的单 `.a` 文件继续有效
- **命名约定 vs 元数据**：平台选择优先用文件名约定（lib/ 层面），`index.toml` 的 platform 用于仓库过滤
