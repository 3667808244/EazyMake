# EazyMake 0.9.4 — 文档与质量完善

---

## 1 背景

0.9.3 完成了捆绑包迁移，项目功能层面已基本稳定。但 temp.md 用户期待分析暴露出若干文档和质量层面的缺口：FAQ/故障排除文档缺失、CHANGES.md 长期未更新、部分错误信息不够友好、Lua API 缺乏版本化机制导致脚本无法做兼容性判断。

本版本聚焦**文档补全**与**代码质量打磨**，不新增核心功能。

---

## 2 目标

1. **FAQ / 故障排除文档** — 新建 `docs/en/faq.md` + `docs/zh/faq.md`，覆盖常见问题和排错流程
2. **CHANGES.md 补全** — 从 0.2.6 补到 0.9.4，补全所有中间版本的变更记录
3. **错误信息打磨** — 清理空异常消息、为核心流程增加 "did you mean" 建议
4. **Lua API 版本化** — 添加 `EZMK_LUA_API_VERSION` 常量 + `ezmk.api_version` Lua 字段，建立向后兼容策略
5. **离线场景文档** — 补上 0.9.3 推迟的离线使用方案

---

## 3 详细设计

### 3.1 FAQ / 故障排除文档

**文件**: `docs/en/faq.md` + `docs/zh/faq.md`

内容结构：
- **安装问题**: MSYS2 环境配置、权限错误、网络问题
- **构建问题**: "找不到头文件"、"链接错误"、"缓存损坏" 的排查步骤
- **包管理问题**: "找不到某某包" → 检查 repo 列表 → 检查网络 → 手动安装
- **配置问题**: `ezmk.toml` 常见错误（字段名拼写、TOML 语法、路径格式）
- **跨平台问题**: Windows 路径分隔符、MSVC vs GCC 标志差异

每条 FAQ 格式：
```
### Q: 构建时报 "fatal error: xxx.h: No such file or directory"
**原因**: ...
**解决**: 1. ... 2. ... 3. ...
```

参考现有文档（`docs/en/@cache.md`、`docs/en/cli.md`）中已有的排错提示，整合到一个集中入口。

### 3.3 错误信息打磨

**待修复项**：

1. **`cli.cpp:65` 空异常消息** — `throw std::invalid_argument("")` → 改为有意义的错误信息（i18n key）
2. **未知 profile 建议** — 当 `--profile <name>` 未匹配时，除列出可用 profile 外，增加 Levenshtein 距离 ≤2 的候选建议（"did you mean: debug?"）
3. **未知命令建议** — 当 `ezmk` 收到未知子命令时，做同样的模糊匹配建议
4. **裸 `std::runtime_error` 审计** — 排查 `src/` 中所有直接 `throw std::runtime_error(...)` 的位置，评估是否改为 i18n 化错误

**实现工具**：在 `util.hpp` 中添加 `closest_match(input, candidates, max_distance=2)` 辅助函数。

### 3.4 Lua API 版本化

**新增 C++ 常量**（`include/ezmk/lua_api.hpp`）:
```cpp
#define EZMK_LUA_API_VERSION 1
```

**新增 Lua 字段**（`src/lua_api.cpp` `register_api()`）:
```lua
ezmk.api_version = 1   -- 整数，递增；脚本可做 if ezmk.api_version >= 2 then ... end
```

**向后兼容策略文档**（在 `docs/en/utils.md` 和 `docs/zh/utils.md` 的 API 参考章节新增一节）：
- API version 为整数，仅在有**不向后兼容的变更**时递增
- 新增函数（向后兼容）不改变 version
- 废弃函数在文档中标记 `@deprecated since v<N>`，保留至少 2 个 minor 版本后移除
- 移除行为触发 version 递增

**范围限制**：本版本仅添加版本号基础设施，不修改任何现有 API 行为。

### 3.5 离线场景文档

在 `docs/en/faq.md` 和 `docs/zh/faq.md` 中增加一节，或在 `docs/en/pkg.md` 末尾补充：

1. **完全离线**：`repo add <local_path> --type local` 添加本地仓库镜像
2. **手动下载归档**：从 GitHub Release 下载 `.tar.gz` → `pkg install <path> --type file`
3. **预置包**：在 USB 或内网共享中部署 `ezmk-repo` 镜像

## 5 兼容性矩阵

| 变更 | 影响 | 处理 |
|---|---|---|
| 新增 `ezmk.api_version = 1` | 旧脚本未使用该字段 | 纯增量，不影响现有脚本 |
| 错误信息文本变化 | 依赖错误消息解析的脚本 | 概率极低；错误消息解析不是稳定接口 |
| `closest_match()` 建议 | 未知命令/profile 输出更友好 | 纯增量 |
| FAQ 文档 | 无 | 新增 |

---

## 6 跨版本关注点

- **向后兼容**: 所有变更为纯增量（新增字段、新增建议），不改变已有行为
- **i18n**: 新增错误信息需同时提供 en/zh 翻译
- **依赖**: 无（本版本独立，可在 0.9.3 完成后随时开始）
