# EazyMake 0.9.7 — 默认仓库生态扩展

---

## 1 背景

0.9.6 完成了依赖版本锁定、构建进度显示等核心功能补全，EazyMake 在功能维度上已接近 1.0.0 标准。但当前默认仓库中的预编译包数量有限，用户开箱即用的体验受限于仓库生态。

本版本聚焦**默认仓库的包扩展**，优先添加社区广泛使用、许可证宽松、构建简单的 C/C++ 库，降低新用户从零配置依赖的门槛。

---

## 2 目标

本版本计划新增以下包（1 个核心包族 + 5 个独立包）：

**独立包**：

| 包名       | 类型            | 许可证       | 复杂度 | 说明                                                     |
| ---------- | --------------- | ------------ | ------ | -------------------------------------------------------- |
| `zlib`     | 压缩库 (C)      | zlib License | ★☆☆    | 广泛使用的压缩库，无数项目的间接依赖                     |
| `cli11`    | CLI 解析 (C++)  | BSD-3-Clause | ★☆☆    | Header-only，单文件                                      |
| `glfw`     | 窗口库 (C)      | zlib/libpng  | ★★☆    | 跨平台 OpenGL/Vulkan 窗口+输入，imgui 最常用平台后端依赖 |
| `sdl2`     | 多媒体库 (C)    | zlib         | ★★★    | 跨平台窗口/音频/输入/游戏手柄，体积较大                  |
| `yaml-cpp` | YAML 解析 (C++) | MIT          | ★★☆    | C++ YAML 解析/生成，CMake 构建                           |

> **注意**: `sdl3` 暂不打包（API 仍在演进，生态尚未成熟，待社区广泛迁移后再评估）。

**`imgui` 包族**（1 个核心 + 16 个后端，详见 §3.2.6）：

| 类别           | 包                                                                                                                                   |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| 核心 (1)       | `imgui`                                                                                                                              |
| 平台后端 (7)   | `imgui-glfw`、`imgui-sdl2`、`imgui-sdl3`、`imgui-win32`、`imgui-glut`、`imgui-osx`、`imgui-android`                                  |
| 渲染器后端 (9) | `imgui-opengl2`、`imgui-opengl3`、`imgui-vulkan`、`imgui-dx9`、`imgui-dx10`、`imgui-dx11`、`imgui-dx12`、`imgui-metal`、`imgui-wgpu` |

> **总包数**: 5（独立） + 1（imgui 核心） + 16（imgui 后端） = **22 个包**。
> 其中 imgui 后端包按优先级分批：第一批完成 GLFW + OpenGL3 + Vulkan + DX11 + Win32（5 个最常用），其余逐步补充。

### 2.1 延后规划（保留）

以下包因复杂度较高或依赖链较深，不在本版本规划范围内，但保留在路线图中供后续版本考虑：

| 包名                       | 延后原因                                                                                                                                                                                 |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `boost` (Header-only 部分) | Boost 体量巨大（100+ 子库），header-only 子库之间也存在内部依赖关系，需逐库验证；各子库版本独立、测试覆盖成本高。建议拆分为多个独立包（如 `boost-asio`、`boost-filesystem`）分版本推进。 |

## 3 详细设计

### 3.1 包目录结构

每个包在默认仓库 `packages/` 下的标准布局：

```
packages/<name>/
├── index.toml        # 包元数据（版本、依赖、平台二进制映射）
├── <version>/
│   ├── include/      # 头文件
│   ├── lib/          # 预编译静态库 (.a / .lib)
│   └── pkg.toml      # 包级配置（可选：工具定义、Lua 脚本等）
```

### 3.2 各包说明

#### 3.2.1 `zlib`

- **版本**: 1.3.1（最新稳定版）
- **构建**: 纯 C，无外部依赖，`./configure && make` 或 CMake
- **产物**: `libz.a` / `zlib.lib` + `zlib.h` / `zconf.h`
- **平台**: Windows (MSYS2/gcc + MSVC)、Linux (gcc/clang)、macOS (clang)
- **注意事项**: MSVC 下需注意 `_WINDOWS` / `WIN32` 宏定义；zlib 1.3.x 要求 C99 编译器

#### 3.2.2 `cli11`

- **版本**: 2.5.0（最新稳定版）
- **构建**: Header-only，无需编译，直接分发头文件
- **产物**: `CLI11.hpp`（单头文件，约 12,000 行）
- **平台**: 跨平台（纯标准 C++11，无平台特定代码）
- **注意事项**: 
  - 虽为 header-only，仍需验证在所有目标编译器下无警告
  - 建议提供 `pkg.toml` 声明 `header_only = true`，使 `ezmk` 跳过编译步骤

#### 3.2.3 `glfw`

- **版本**: 3.4（最新稳定版）
- **许可证**: zlib/libpng（非常宽松，允许静态链接、修改、再分发）
- **构建**: CMake，纯 C99，无外部依赖（直接调用系统 API）
- **产物**: `libglfw3.a` + `include/GLFW/`
- **平台**: Windows (MSYS2/gcc + MSVC)、Linux (gcc/clang)、macOS (clang)
- **Linux 注意事项**: 编译时需要 X11/Wayland 开发头文件（`libx11-dev`、`libxkbcommon-dev`、`wayland-protocols` 等），但**运行时仅依赖系统已安装的 `.so`**，不增加用户运行时负担。可在 `pkg.toml` 中声明 `system_deps_linux = ["x11", "xkbcommon", "wayland"]` 供 `ezmk` 做友好提示

#### 3.2.4 `sdl2`

- **版本**: 2.30.x（最新稳定版）
- **许可证**: zlib（与 GLFW 相同，非常宽松）
- **构建**: CMake，纯 C99
- **产物**: `libSDL2.a` + `include/SDL2/`
- **平台**: Windows (MSYS2/gcc + MSVC)、Linux (gcc/clang)、macOS (clang)
- **Linux 注意事项**: 编译时依赖 X11/Wayland/PulseAudio 等子系统头文件，但均为系统级开发包（`apt install libsdl2-dev` 即可一键安装）；运行时依赖系统 `.so`。子系统可按需裁剪（如 `-DSDL_AUDIO=OFF`），但默认全功能编译
- **体积**: 全功能编译约 3-5 MB（比 GLFW 大一个数量级），首次 `pkg install` 下载时间较长

#### 3.2.5 `yaml-cpp`

- **版本**: 0.8.0（最新稳定版）
- **构建**: CMake，依赖 C++11
- **产物**: `libyaml-cpp.a` + `include/yaml-cpp/`
- **平台**: Windows (MSYS2/gcc + MSVC)、Linux、macOS
- **注意事项**:
  - 0.8.0 需要 CMake 3.10+；Windows 下可用 CMake + Ninja 或 MSVC 直接编译
  - 模板较重，编译时间较长（~2-5 分钟单线程），考虑预编译各平台二进制以加速用户安装

#### 3.2.6 `imgui` + 后端全家桶

Dear ImGui 打包分为 **1 个核心包** + **N 个后端包**。每个后端包独立安装，用户按需组合（1 个平台后端 + 1 个渲染器后端）。

**核心包 `imgui`**：
- **版本**: 1.91.9
- **构建**: 编译 `imgui/` 下核心 `.cpp` 文件（约 10 个），不含 `backends/`
- **产物**: `libimgui.a` + `include/imgui/`（核心头文件 + 源文件）
- **依赖**: 无外部依赖（纯标准 C++11）

**后端包一览**（每个包 = 1 个 `.h` + 1 个 `.cpp`，依赖 `imgui` 核心）：

| 包名            | 类型   | 跨平台         | 额外系统依赖          | 说明                               |
| --------------- | ------ | -------------- | --------------------- | ---------------------------------- |
| `imgui-glfw`    | 平台   | ✅ Win/Lin/Mac  | `glfw`（EZMK 包）     | 最流行的窗口后端                   |
| `imgui-sdl2`    | 平台   | ✅ Win/Lin/Mac  | `sdl2`（EZMK 包）     | SDL2 窗口后端                      |
| `imgui-sdl3`    | 平台   | ✅ Win/Lin/Mac  | `sdl3`（延后）        | SDL3 窗口后端（依赖未就绪）        |
| `imgui-win32`   | 平台   | ❌ Windows only | 无（系统 API）        | Windows 原生窗口                   |
| `imgui-glut`    | 平台   | ✅ Win/Lin/Mac  | `freeglut`            | 传统 GLUT 窗口（教学/简单场景）    |
| `imgui-osx`     | 平台   | ❌ macOS only   | 无（Cocoa）           | macOS 原生窗口（Obj-C）            |
| `imgui-android` | 平台   | ❌ Android only | Android NDK           | Android 原生窗口                   |
| `imgui-opengl2` | 渲染器 | ✅ Win/Lin/Mac  | `libGL` / `opengl32`  | 传统 OpenGL 2.x（兼容旧硬件）      |
| `imgui-opengl3` | 渲染器 | ✅ Win/Lin/Mac  | `libGL` / `opengl32`  | OpenGL 3.2+ Core Profile（最常用） |
| `imgui-vulkan`  | 渲染器 | ✅ Win/Lin/Mac  | `vulkan` SDK          | Vulkan 渲染                        |
| `imgui-dx9`     | 渲染器 | ❌ Windows only | `d3d9`（系统）        | DirectX 9（兼容旧硬件）            |
| `imgui-dx10`    | 渲染器 | ❌ Windows only | `d3d10`（系统）       | DirectX 10                         |
| `imgui-dx11`    | 渲染器 | ❌ Windows only | `d3d11`（系统）       | DirectX 11（Windows 主流）         |
| `imgui-dx12`    | 渲染器 | ❌ Windows only | `d3d12`（系统）       | DirectX 12                         |
| `imgui-metal`   | 渲染器 | ❌ macOS/iOS    | 无（Metal.framework） | Apple Metal 渲染（Obj-C）          |
| `imgui-wgpu`    | 渲染器 | ✅ Win/Lin/Mac  | `wgpu-native`         | WebGPU（新兴标准，浏览器+原生）    |

**后端包结构**（以 `imgui-glfw` 为例）：
```
packages/imgui-glfw/
├── index.toml
├── 1.91.9/
│   ├── include/
│   │   └── imgui/
│   │       └── backends/
│   │           └── imgui_impl_glfw.h
│   ├── lib/
│   │   └── libimgui_glfw.a
│   └── pkg.toml        # depends = ["imgui", "glfw"]
```

**用户使用方式**：
```toml
# ezmk.toml — 选择 GLFW + OpenGL3 组合（最常用）
[depends]
lib = ["imgui", "imgui-glfw", "imgui-opengl3"]
```
```cpp
// main.cpp
#include "imgui/imgui.h"
#include "imgui/backends/imgui_impl_glfw.h"
#include "imgui/backends/imgui_impl_opengl3.h"
```

**注意事项**:
- 每种后端仅编译 `.cpp` 源文件为 `.a`，头文件路径统一放在 `include/imgui/backends/` 下
- `imgui-glfw` → 依赖 EZMK 包 `glfw`；`imgui-sdl2` → 依赖 EZMK 包 `sdl2`；安装时 `ezmk` 自动解析传递依赖
- 仅依赖系统 API 的后端（`imgui-win32`、`imgui-osx`、`imgui-metal`）无额外依赖，用户体验最简
- `vulkan` SDK 和 `wgpu-native` 暂不作为 EZMK 包（体积大、平台差异大），用户通过系统包管理器安装
- Obj-C 后端（`imgui-osx`、`imgui-metal`）仅在 macOS 上编译，其他平台安装时跳过
- `imgui-sdl3` 标记为延后（`sdl3` 未打包）；`imgui-dx9`/`dx10` 低优先级（用户稀少）
- 优先级：先完成 **GLFW + OpenGL3 + Vulkan + DX11 + Win32** 五个最常用后端，其余逐步补充

### 3.3 包验证标准

每个包在上线前需通过以下验证：

1. **编译验证**: 在 Windows (MSYS2 gcc)、Linux (gcc)、macOS (clang) 三个平台上成功编译
2. **链接验证**: 在示例项目中使用 `ezmk pkg install <name>` 安装后，`project build` 成功链接
3. **头文件完整性**: `#include` 包的主要头文件，无缺失依赖
4. **版本信息准确**: `index.toml` 中的版本号、依赖声明与实际一致

| ---------------------------- | -------------------------------------- | ----------------------------------------------------------------------- |

## 6 跨版本关注点

- **依赖**: 依赖 0.9.6（依赖版本锁定功能就绪后，可为新包声明精确版本约束；`imgui-glfw` → `glfw` → 无依赖，传递链清晰）
- **向后兼容**: 纯增量变更，不影响现有用户
- **1.0.0 准备**: 丰富默认仓库生态是 1.0.0 发布前的最后一个关键里程碑；本版本完成后，默认仓库将覆盖：日志/测试/JSON/压缩/CLI/窗口/多媒体/YAML/GUI（含全后端）等常见领域
- **延后项**: `boost` header-only 包的拆分方案、`sdl3` 包（待生态成熟）、`vulkan`/`wgpu` SDK 包（体量/复杂度高）留待后续版本
