# 安全性

## 包安装

- 在安装包时，如果使用全局安装，必须二次确认
- 在安装包时，如果会覆盖已有文件，必须二次确认
- 从 URL 下载包时，若仓库 `index.toml` 提供了 `sha256`，必须校验

## 仓库管理

- `git clone` 失败时，清除不完整的本地缓存目录后报错
- `git pull` 失败时，发出警告但保留已有缓存继续运行（不阻断构建）
- 全局作用域的仓库注册本身无需二次确认（clone 不等同于安装）
- 但通过全局仓库安装包时仍触发全局安装确认（见上方包安装规则）

## 缓存

- 写入 `record.json` 和 `.o` 文件时先写 `.tmp` 临时文件，成功后 `rename` 覆盖
- 构建中途失败不会破坏已有缓存的一致性
