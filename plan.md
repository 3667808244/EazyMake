## 一、当前版本：0.9.5.1 — 代码重构与质量清理

> 详细设计: [`plans/0.9.5.1.md`](plans/0.9.5.1.md)

### 1.1 背景

0.9.5 完成了跨平台体验与质量保障（install.ps1 + 集成测试 + 三平台冒烟），在 0.9.6 开始新功能之前，进行一次集中的代码质量清理。**本版本不新增用户可见功能**，仅做代码级清理。

### 1.2 四维度清理目标

| 优先级 | 类别 | 具体工作 | 收益 |
|--------|------|----------|------|
| **P0** | 代码重复消除 | `link_phase()` 6 个重复块提取为 `execute_link()`；`compile_sources()` 改为调用 `compile_one_source()`；debounce 刷新统一为 `check_and_flush()`；main.cpp 仓库自动更新去重；config.cpp 配置名校验去重 | 减少约 600+ 行重复 |
| **P0** | 资源管理修复 | `g_cached_config` → `unique_ptr`；`OVERLAPPED*` → 内嵌值对象 | 消除泄漏风险 |
| **P1** | 测试盲区补全 | `compare_version()` 15+ 场景测试；`extract_archive()` 基础测试；依赖解析边界测试；编译选项签名测试；提取共享 fixtures 到 `test/test_helpers.hpp` | 核心路径覆盖 |
| **P2** | 魔术字符串集中化 | 引入 `ezmk::path::*` 常量（`.ezmk`/`.ezmk/cache`/`.ezmk/pkg` 等），逐文件迁移 | 统一路径管理 |
| **P2** | 死代码移除 | 移除 `ParsedOptions::count()`、`native_path()` | 减少维护负担 |
| **P3** | 头文件与 i18n 清理 | `file_watcher.hpp` 复用 `util.hpp` 平台宏；toolchain/cache 硬编码字符串 → i18n key | 代码整洁 |

### 1.3 当前进度与下一步

执行分四个阶段（代码重复消除 → 资源管理修复 → 测试补全 → 清理收尾），每阶段完成后运行全量测试确认无回归。

**核心原则**：所有变更必须保持与 0.9.5 行为一致。重构前后均运行全量测试（491 测试用例、2250 断言）。

## 二、下一版本：0.9.6 — 功能补全与生态完善

> 详细设计: [`plans/0.9.6.md`](plans/0.9.6.md)

### 2.1 四项交付

| # | 交付物 | 说明 |
|---|--------|------|
| 1 | **依赖版本锁定** | `ezmk.toml` 中 `[depends]` 支持 `foo@1.2.3` / `^1.0` / `~1.2` / `>=1.0` 版本约束语法 |
| 2 | **构建进度显示** | 并行编译时显示 `[N/M] src/file.cpp` 逐文件进度 + 构建完成摘要 |
| 3 | **.clang-format 基础设施** | 项目根目录添加 `.clang-format` 配置 + `CONTRIBUTING.md` 代码风格章节 |
| 4 | **ASCII Logo** | `ezmk` 裸运行显示项目 Logo（彩色、与 `--color` 联动） |

### 2.2 关键设计决策

- **版本锁定**：不引入锁定文件（`ezmk.lock`），留待 1.0.0 之后。向后兼容纯字符串格式（无约束）。
- **构建进度**：仅在 `-j > 1` 或 `--verbose` 时显示逐文件进度；单文件编译保持简洁。
- **.clang-format**：不添加 `ezmk utils format` 命令，仅提供配置文件。
- **依赖**：依赖 0.9.5.1（代码清理完成后，重构后的清晰代码更易扩展）。

### 2.3 里程碑意义

0.9.6 完成后，EazyMake 在**功能、文档、质量、跨平台**四个维度上达到 1.0.0 标准。剩余版本（0.9.7~0.9.9）聚焦生态扩展和技术债务消除。

> 详细设计: [`plans/0.9.9.md`](plans/0.9.9.md)

消除最后的技术债务——安装钩子从 Shell 脚本迁移至 Lua：

| 变更 | 说明 |
|------|------|
| `.lua` 脚本优先 | `detect_install_script()` 优先返回 `.lua`（跨平台统一） |

| 向后兼容 | 旧包的 `.sh`/`.ps1`/`.bat` 作为 fallback 继续工作 |

**里程碑意义**：0.9.9 完成后，项目实现"零技术债务"——构建钩子和安装钩子统一使用 Lua + sandbox。

## 五、跨版本关注点

### 5.1 向后兼容性

- `ezmk.toml` 格式扩展（版本约束语法）保持旧格式兼容
- CLI 接口稳定——所有新增 flag 为纯增量
- 0.9.5.1 的重构必须保持二进制等价行为

### 5.2 安全模型

- Lua sandbox 贯穿所有钩子系统（构建钩子 0.2.3 + 安装钩子 0.9.9）
- SHA-256 校验（下载 + 安装全链路）
- Lua 安装钩子无需编辑器审查（sandbox API 边界已限定）

### 5.3 测试策略

- 单元测试：491 用例 / 2250 断言（0.9.5.1 补全盲区后更多）
- 集成测试：4 个端到端场景（0.9.5 引入，`[integration]` tag）
- 冒烟测试：三平台手动验证（Linux/macOS/Windows）
- 每版本执行 `build.sh test-all` 确认无回归

### 5.4 文档一致性

- `docs/en/` ↔ `docs/zh/` 一一对应（0.9.2 建立）
- 1.0.0 补全 `docs/zh/cli.md` 翻译（最后一块缺失的拼图）
- 新增功能需双语同步交付
- `plans/` 目录在 1.0.0 完成拆分（`dev/` + `release/`）
